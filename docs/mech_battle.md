# Бой с мехом (EnemyMech)

Документ описывает концепцию дуэли, поведение вражеского меха, его боевой
паттерн и механику рывка (дэш) — у меха и у башни-игрока.

Код: [`scripts/enemy_mech.gd`](../scripts/enemy_mech.gd) (наследник
`SkeletonArcher`), визуал рывка [`scripts/dash_fx.gd`](../scripts/dash_fx.gd),
дэш башни — [`scripts/tower.gd`](../scripts/tower.gd).

---

## 1. Концепция: соло-apex дуэль

- **Башня — это «гномий мех» (APEX).** Противостоять ей по-настоящему может
  только другой мех. Скелеты/гиганты — это давление толпы; мех — это **личная
  дуэль** игрока (управляющего башней) против равного.
- **Мех целит ТОЛЬКО башню.** Лагерь, гномов, постройки он игнорирует полностью
  (`_resolve_target` возвращает только `Tower`). Роль чистая: лагерь — забота
  защитников, мех — угроза лично башне.
- **Тематика.** Механический конструкт: холодный металл + emission-«реактор»
  (голубое свечение), отличает его от нежити-скелетов и багряных гигантов.
- **В волнах — строго ОДИН за раз.** Дизайн-правило: два меха одновременно
  непротягиваемы (две дуэли разом игрок не вывозит). Сейчас спавнится **только
  читом** `WaveDirector.cheat_spawn_mech`; авто-спавн в волнах + мех-оружие
  башни + финальный баланс дуэли — следующий этап.

## 2. Базовые статы (scenes/enemy_mech.tscn)

| Параметр | Значение | Смысл |
|---|---|---|
| `hp` | 400 | Толстый — дуэль длится, не сносится парой залпов |
| `move_speed` | 7.0 | Быстрый, подвижная цель |
| `vision_radius` | 60 | Видит башню издалека |
| `attack_radius_min / max` | 14 / 24 | Полоса, в которой мех держит дистанцию (стрейф) |
| `attack_windup` | 1.0 c | Замах (телеграф) перед выстрелом |
| `attack_cooldown` | 2.0 c | Базовый откат лучника |

Поверх архетипа лучника (kite-FSM: подойти → замах-телеграф → выстрел → откат)
мех добавляет хореографию по дистанции, набор атак и реактивность.

## 3. Дистанционная хореография (3 полосы + буфер)

Мех ведёт себя по-разному в зависимости от дистанции до башни. Полосы заданы
`kite_min_range` (22 м) и `kite_max_range` (50 м):

```
 0 ──── 10 ──────── 22 ─────────────── 50 ──────► дистанция до башни
 │ ОТБРОС │ БЛИЖНЯЯ │     СВЯЗКА        │  СПРИНТ
 │(shock) │ AIMED↔  │  Поле → Ракеты    │ (догоняет
 │        │ Шквал   │  «поймал→добил»   │  pursuit×1.7)
```

- **Упор (< `shock_trigger_range` = 10 м) — мгновенный ОТБРОС.** Мех
  дальнобойный, в ближний бой не лезет. Подполз вплотную → **без замаха и
  телеграфа** бьёт ударной волной: урон по башне + сильный отброс ПРОЧЬ +
  глушит управление. Это **наказание за зажим**, а не игра в уворот. Кулдаун
  `shock_cooldown` (5 c). См. `_tick_shock` / `_burst_shock`.
- **Ближняя фаза (< 22 м) — чередование AIMED ↔ Шквал.** Читаемый «джеб-ритм»
  (детерминированно, не рандом). См. `_pick_attack`.
- **Боевая полоса (22–50 м) — связка Поле → Ракеты.** «Поймал замедлением →
  добил залпом». См. `_tick_kite_combo`. Здесь же изредка прилетает
  парируемый **супер-залп**.
- **Дальше 50 м — спринт-преследование.** Скорость ×`pursuit_speed_mult` (1.7),
  чтобы не было «безопасной парковки» за зоной связки — мех догоняет.
- **Стрейф (облёт).** В фазах замаха/отката мех не стоит столбом (как базовый
  лучник), а **кружит** вокруг башни в полосе [14, 24] м (`_strafe_around`),
  оставаясь подвижной целью. Направление меняется каждые `strafe_flip_interval`
  (2.5 c), чтобы круг не был предсказуемым.
- **Обтекание препятствий.** Локальный сфера-запрос (`_apply_obstacle_avoidance`)
  отруливает меха от палаток/ресурсов (`AVOID_MASK`), чтобы он огибал их, а не
  таранил. Башню и стены не избегает (башня — цель).

## 4. Паттерн: бит-дирижёр

Чтобы три подсистемы атак (ближняя / связка / супер) не сливались в кашу, есть
**глобальный «бит»** — общий «вдох» после ЛЮБОЙ атаки:

- `global_attack_cooldown` (3 c) — пока он тикает, **никто не атакует**.
  Ближний замах растягивается минимум до конца паузы; связка ждёт. Это
  сериализует всё в один читаемый ритм **«телеграф → атака → вдох»** —
  главный рычаг плотности боя.
- **Frenzy (окно наказания).** Пока башня **поймана замедлением поля**, бит
  умножается на `frenzy_beat_mult` (0.45 → почти вдвое чаще), и мех **долбит
  ракетами подряд** без перезахода в поле. Это и есть «достреливание пойманного».
  См. `_beat()` и `_target_slowed`.
- **Рваный ритм.** Длительности замаха/отката джиттерятся на ±`rhythm_jitter`
  (35%), чтобы игрок не затаймил свои уклонения наизусть.

## 5. Арсенал атак

Все снаряды меха — **тот же `fireball.tscn`, что кастует башня** («атака меха =
атака башни»), но с хостайл-маской `MECH_AOE_MASK` (бьёт башню/лагерь/стены/
гномов **И** скелетов через `ENEMIES`). В полёте снаряд НЕ рвётся о скелетов
(чтобы долетал до прицела), но AOE на импакте задевает ближних.

| Атака | Где | Что делает |
|---|---|---|
| **AIMED** | ближняя фаза | Один прицельный фаербол в упреждённую точку |
| **Шквал (SPREAD)** | ближняя фаза | Веер из `spread_count` (4) фаерболов, разнесённых перпендикулярно линии огня (`spread_spacing` 6 м). Развёртка по одному с интервалом `spread_interval` (0.22 c). **Честность:** просветы между взрывами есть — ловишь максимум один, между ними щель для рывка |
| **Поле (slow-field)** | связка, 22–50 м | Пузырь-капкан летит на игрока с homing'ом; догнал → лопается зоной (`field_radius` 8 м), внутри башня **замедляется** (`field_slow_factor` 0.45 — и ходьба, и рывок). Урона НЕ наносит — это **сетап** под ракеты. «Ловим мобильность дебафом, а не гонкой урона» |
| **Ракеты вдогонку** | связка / frenzy | Залп `missile_count` (3) слабо-самонаводящихся ракет. Anti-kite: достают башню, когда она кайтит за обычной дальностью. Слабый `missile_turn_rate` — **рывок СБИВАЕТ ракету** (homing не должен решать сам) |
| **Супер-залп** | редкий, 22–50 м | Рой `missile_super_count` (6) ракет **в одну точку**, длинный телеграф (`missile_super_warn` 1.3 c) + длинный кулдаун (14 c). Ракеты **Reflectable** — поймал окном парирования (Q) → весь рой летит обратно в меха (жирный урон). Не отбил — съел весь залп |
| **Отброс (shockwave)** | упор < 10 м | Мгновенный панишинг за зажим (см. §3) |

## 6. Телеграфы и упреждение

Мех **честно показывает, куда ударит**:

- **Живой телеграф.** В фазе замаха на земле — кольцо-«tell» размером с реальный
  AOE, которое **едет за упреждённой точкой** (`_update_telegraph`). Игрок видит,
  как точка удара «ведёт» за движущейся башней, и может **сорвать прицел** резкой
  сменой курса.
- **Упреждение (`aim_lead` 0.9).** Точка = позиция башни + её скорость × время
  полёта (с капами `aim_lead_max_time` и `aim_lead_max_distance` 4 м, чтобы не
  улетала в небо). Бьёт по равномерно движущейся башне, но переигрывается манёвром.
- **Lock (коммит).** За `telegraph_lock_time` (0.35 c) до выстрела точка
  **замирает и пульсирует** тревожным цветом. Это **окно уворота**: видишь
  финальную точку — уходишь.

## 7. Реактивность: «умнее скелетов»

Главное отличие меха от скелета — он **уходит с линии огня**:

- **Скан угрозы.** Каждый кадр ищет ближайший летящий фаербол игрока (группа
  `player_projectile`) в радиусе `dodge_detect_radius` (9 м). Фаербол игрока
  homing'ит в зафиксированную точку каста (не трекает меха), поэтому уклонение
  реально уводит с траектории. См. `_scan_threat`.
- **Evade-рывок (`_start_evade`).** Уход **перпендикуляром** к линии угрозы (+чуть
  прочь от снаряда), сторона выбирается так, чтобы **не нырнуть под башню**.
  Кулдаун `dodge_cooldown` (1 c).
- **Пуниш (контратака).** После уворота: следующий замах укорачивается до
  `snap_windup` (0.3 c, **снап-выстрел**), а текущий откат срезается до
  `punish_cooldown` (0.4 c). Эффект «увернулся И сразу огрызнулся».

## 8. Дэш (рывок)

Дэш — общий механизм и визуал у меха и у башни (см. `DashFx`: наклон вперёд +
вытягивание вдоль рывка + after-image-трейл). По длине рывки **идентичны**:
`dash_speed` × `dash_duration` = 22 × 0.24 ≈ **5.3 м**.

### 8.1. Дэш меха (evade)
- Не управляемый игроком — это **реакция уклонения** (см. §7). Срабатывает на
  летящий фаербол, уводит меха перпендикуляром с линии огня.
- Активный рывок **перекрывает** обычное движение (kite/стрейф): пока
  `_dash_remaining > 0`, скорость = `_dash_vec`, остальная логика движения в этом
  кадре пропускается.
- Реализация: `_dash_vec` / `_dash_remaining`, тик в `_ai_step`, визуал в
  `_process` через `DashFx`.

### 8.2. Дэш башни/игрока (клавиша Space)
[`scripts/tower.gd`](../scripts/tower.gd), группа экспортов «Dash (рывок, Space)».

- **Направление:** текущее направление движения, а если стоишь — последнее
  (`_last_move_dir`). Нельзя дэшнуть «в никуда».
- **Скорость/длина:** `dash_speed` 22 м/с, `dash_duration` 0.24 c (≈ 5.3 м),
  кулдаун `dash_cooldown` 0.8 c.
- **Урон тараном (`dash_damage` 100).** Врезался рывком в скелета — задавил
  (100 убивает обычных скелетов с hp=30 сразу; гигантов лишь дробит). Каждого
  скелета бьёт **раз за рывок** (`_dash_hit_set`, без мультихита). **Мех
  тараном НЕ давится** (как и щитом) — против него дэш это инструмент уклонения,
  а не урона.
- **Взаимодействие с дебафами:**
  - **Slow-field меха** скейлит И ходьбу, И рывок (`slow` = `_slow_factor` 0.45):
    рывок **ослаблен** (короче), но не выключен — пойманный игрок всё ещё может
    дёрнуться, просто недалеко.
  - **Knockback от отброса меха** перебивает всё: пока активен, ввод/рывок
    игнорируются, рывок не «копится» под отбросом (`_dash_timer` тикает в ноль).
- **Роль в дуэли:** дэш — основной способ уйти из-под зафиксированного телеграфа
  (§6), сбить самонаводящуюся ракету (§5) и выскочить из щели в Шквале. Парирование
  (Q) — против ракет/супера; дэш — против прицельных и площадных.

## 9. Парирование (Q) — контр-инструмент против снарядов

[`scripts/tower.gd`](../scripts/tower.gd), группа экспортов «Parry (парирование, Q)».
Парирование — **второй столп обороны** наряду с дэшем: дэш убирает башню с
траектории, парирование **разворачивает снаряд обратно в меха**.

- **Тайминг-окно.** По нажатию Q открывается короткое окно `parry_window`
  (0.25 c) — чистый скилл, **маны не требует**, гейт только по кулдауну
  `parry_cooldown` (1.5 c). Короткое окно награждает **реакцию на телеграф/подлёт**,
  а не зажатие кнопки. См. `_try_start_parry`.
- **Зона ловли.** Купол радиусом `parry_radius` (6 м, 360° вокруг точки каста).
  Пока окно активно, КАЖДЫЙ вражеский снаряд (`Reflectable.GROUP`) в радиусе
  **разворачивается обратно в стрелка и становится дружественным**
  (`Reflectable.try_reflect` в `_tick_parry`).
- **Что отбивается.** Всё, что мех регистрирует как `Reflectable`: обычные
  фаерболы/Шквал, ракеты вдогонку и — ключевое — **рой супер-залпа**. Снаряд
  помнит стрелка (`set_shooter(self)`), поэтому отражённый летит **в меха, не в
  скелета**, и наносит ему урон (мех считает его в телеметрии через
  `note_reflected_damage`).
- **Бонус: щит бьёт скелетов.** На активации купол наносит `parry_skeleton_damage`
  (100) скелетам в зоне (разовый импульс, не тик) — мех/башню/гномов не трогает.
- **Риск/награда супера.** Супер-залп специально кучный (`missile_super_interval`
  0.05 c) — весь рой прилетает плотно, **одно окно Q ловит весь залп** → жирный
  отражённый урон в меха. Не отбил — съел 6×40 урона. Длинный телеграф
  (`missile_super_warn` 1.3 c) даёт время прочитать и приготовить Q.

**Разделение труда:** парирование — против **снарядов** (фаербол/ракеты/супер);
дэш — против **прицельных и площадных** (зафиксированный телеграф, щель в Шквале,
сбить homing ракеты манёвром); отброс меха в упор парированием НЕ отбивается
(это не снаряд) — от него спасает только не-зажимать дистанцию.

## 10. Анатомия одного цикла связки (потактово)

Пример: башня кайтит на ~35 м (боевая полоса 22–50). Мех ведёт связку
**Поле → Ракеты**, бит-дирижёр сериализует шаги. `T` — секунды.

```
T=0.0   Шаг 1 связки: пущен темпоральный заряд (пузырь-капкан) в башню.
        Ставится общий бит (вдох) = global_attack_cooldown (3.0 c). _combo_step→1.
        ──► реакция игрока: пузырь летит дугой с homing'ом → УВЕСТИ рывком
            (turn у заряда низкий, дэш ≈5.3 м сбивает захват).

T≈0..3  «Вдох»: мех НЕ атакует, кружит-стрейфит вокруг башни (подвижная цель),
        может уклониться evade-рывком, если игрок стреляет в него.

  ── развилка по исходу поля ──
  A) Пузырь ДОГНАЛ → лопнул slow-зоной (r=8 м). Башня поймана:
        _target_slowed=true → FRENZY: бит ×0.45 (≈1.35 c), и мех НЕ
        перезаходит в поле, а ДОЛБИТ ракетами подряд («достреливание»).
        ──► реакция: рывок ослаблен полем (×0.45) — выходить из зоны заранее,
            парировать прилетающие ракеты (Q).
  B) Пузырь увернулся / погас за lifetime (5 c): _target_slowed=false.

T≈3.0   Шаг 2 связки (коммит на следующем бите): залп ракет (3 шт,
        live-homing к башне). Ставится бит (3.0 c, или frenzy ×0.45, если A).
        ──► реакция: рывок СБИВАЕТ слабый homing ракет; либо парировать (Q) —
            отразить обратно в меха.

T≈6.0   Связка завершена → пауза kite_combo_cooldown (3.0 c), затем заново с T=0.

  ── поверх связки, асинхронно ──
  • Если игрок ВЛОМИЛСЯ < 22 м  → связка глохнет, включается ближняя фаза
    (AIMED↔Шквал, чередование через бит).
  • Если подполз < 10 м         → МГНОВЕННЫЙ отброс (без телеграфа) + урон,
                                   кулдаун 5 c.
  • Изредка (кулдаун 14 c) вместо обычного — СУПЕР-залп (6 ракет в точку,
    телеграф 1.3 c): окно для Q на весь рой.
  • Дальше 50 м                  → мех спринтует (×1.7), связка ждёт дистанции.
```

Ключевая идея ритма: **в любой момент активна максимум одна атака** (бит-дирижёр),
поэтому у игрока всегда есть читаемое окно на реакцию — рывок или парирование.

## 11. Смерть меха

Мех не просто рассыпается — он **детонирует** (`_on_destroyed`):
- Взрыв корпуса (`AoeVisual.spawn_explosion`, радиус `death_explosion_radius` 6 м)
  + расходящаяся ударная волна (`death_shockwave_radius` 9 м).
- **Урон по СКЕЛЕТАМ** в радиусе (`death_explosion_damage` 120) + отброс — смерть
  меха **расчищает толпу вокруг** (награда за победу в дуэли). Башню и гномов НЕ
  задевает (`DEATH_AOE_MASK` = только `ENEMIES`).
- FAR-LOD скелеты (вне broad-phase) добираются group-скан'ом.

---

## 12. Телеметрия (дев-инструмент)

При `telemetry_enabled` мех логирует поведение башни в бою и печатает сводку по
смерти/despawn'у: распределение по дистанциям (вплотную/полоса/кайт), движение и
курс (убегает/сближается/боком), доля укрытия (LoS перекрыт), напор (снаряд
рядом), потерянное башней HP, нагрузка (атак/мин, средний промежуток) и сколько
урона мех получил **отражением** (окупается ли парирование). См. `_telemetry_*`.

## 13. Сводка ключевых параметров

| Группа | Параметр | Значение |
|---|---|---|
| Бит | `global_attack_cooldown` | 3.0 c |
| Бит | `frenzy_beat_mult` (пока башня в поле) | 0.45 |
| Полосы | `shock_trigger_range` / `kite_min` / `kite_max` | 10 / 22 / 50 м |
