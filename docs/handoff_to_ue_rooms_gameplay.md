## 1. Обзор нового геймлупа

Пивот: вместо открытой карты с караваном — **подземелье из комнат-анфилады**. Игрок ведёт **мобильную башню** через комнаты, управляя **рукой-курсором**, командует **отрядами гномов** (копейщики / лучники / рабочие), кастует магию и **строит укрепления**.

**Сюжетная петля (цель — победа матча):**
1. Исследуешь комнаты, освобождаешь поселения гномов.
2. Гномы-строители (Room6) дают квест: **научись строить** — запусти станок-чертёжник в Room11 (пазл).
3. Получив знание построек, можешь возводить стены/башни.
4. Убиваешь **камнеметателя** (босс-враг) → строители вооружают тебе **лучников**.
5. Восстанавливаешь **нефтекачалку** (Room8) — большой механизм у поселения.
6. Качалка добывает «гномью нефть»; ты **защищаешь** её от штурма.
7. **Накопил N нефти → победа.** (Нефть = условие победы матча.)

**Ощущение:** «веди живую крепость через враждебное подземелье, обустраивай плацдарм, держи оборону у добывающего механизма».

## 2. Ключевые акторы (управление)

### 2.1 Башня (Tower) — мобильная крепость игрока
Управляется напрямую (WASD-подобно). HP высокий (≈1000). Ключевые действия:
- **Движение** по физике (не по навмешу) — упирается в стены/препятствия.
- **Дэш** (рывок): урон скелетам при контакте (`dash_damage≈100`), кулдаун. См. §6.6.
- **Супер-дэш** (зажать): мощнее (`super_dash_damage≈250`), тратит ману, ломает двери-`room_door`, бьёт «тяжёлых» врагов (см. §6.6), слоумо на старте.
- **Парирование (щит)** — клавиша Q: окно ≈0.25с, кулдаун, **3 заряда подряд** затем длинная перезарядка. Отражает вражеские снаряды обратно; импульс-урон скелетам в радиусе купола.
- **Мана**: регенерится медленно; пополняется орбами с убитых врагов; тратится на магию/супер.
- **Смерть башни** = взрыв (AoE-урон вокруг) + поражение матча.
- **Зона сбора** (`gather_radius≈16м`) — видимый диск вокруг башни; в нём рабочие добывают/строят (см. §4).

### 2.2 Рука (Hand) — курсор-манипулятор
Парящая рука у курсора. Делает:
- **Хватать/подсвечивать** (ЛКМ `hand_grab`): pickup-объекты (колокол, рычаги, ящики) подсвечиваются при наведении.
- **Кастовать магию**: Искра, Фаербол и т.д. (см. старый §6).
- **Ставить постройки**: тащит силуэт здания → ставит на точку (§5).
- **Командовать отрядами**: клик по земле/области — приказ отряду (идти/собирать/строить).
- **Тянуть рычаги** (пазлы, §3).

### 2.3 Отряды гномов
Покупаются в гномьих домах за золото. Три типа:
- **Копейщики** (melee squad) — мобильный бой.
- **Лучники** (archer squad) — дальний бой; теперь продаются строителями после убийства камнеметателя (§8).
- **Рабочие** (worker artel) — добыча и стройка (§4).

## 3. Уровень: комнаты-анфилада

Комнаты идут цепочкой; башня едет из одной в другую через проёмы/двери. Внутри — скелеты, ресурсы, кувшины с золотом, гномьи дома, пазлы.

### 3.1 Двери
- **RoomDoor** — мягкий барьер-проём. Ломается **ТОЛЬКО супер-дэшем** башни (обычный дэш/магия — нет). На слом: разлетается осколками, открывает проход, перестраивает навигацию.
- **MetalDoor** — металлическая дверь, открывается **активацией механизма** (искрой по диоду): съезжает вниз в пол.

### 3.2 Гном-только проходы
Узкие лазы: **гном пролезает, башня — нет**. Используются в пазлах/лабиринтах (Room10) — игрок шлёт гнома туда, куда сам проехать не может.

### 3.3 Пазлы «ток» (искра → диод → провод → рычаг → дверь)
Единый язык механизмов (UE-команде это пригодится для нефтекачалки):
- **Искра** (заклинание руки) — зигзаг-снаряд; в радиусе импакта оповещает все «spark-цели».
- **Синий диод (SparkDiode)** — на попадание искрой загорается (синий→зелёный) и зовёт `activate()` цели (дверь или красный диод).
- **Красный диод (RedDiode)** — на активацию гонит «бусину тока» по процедурному проводу к цели; на прибытии **запитывает рычаг**.
- **Рычаг (Lever)** — активируется **рукой** (схватить у основания, протянуть вбок до фиксации) ИЛИ **гномом** (бьёт по нему, если рычаг настроен на гнома). Включённый рычаг зовёт `activate()` цели (обычно дверь). Гейт `gnome_needs_power` держит рычаг мёртвым до запитки током.

## 4. Гномы-рабочие (Workers) — добыча и стройка

**Визуал:** бурый гном без копья. Покупается у гномьего дома («Нанять артель рабочих»), отряд из 3.
**Базовые параметры:** hp ≈60, урон по дереву/стройке 4–6 (НЕ по врагам), скорость ≈2.4.

### 4.1 Принцип «гном → точка → действие»
Рабочий не воюет. Он **заряжается на цель и бьёт страйком** (как по кувшину/рычагу) — рубит дерево, носит бревно, строит. Враждебных целей не ищет, боевого выпада нет.

### 4.2 Курьерская петля (видимая)
Флаг «несёт бревно» определяет, какая цель его примет:
- **Руки пусты** → идёт к ресурсу (дерево/камень/железо), рубит, получает «бревно» (на руках видимая коробка-груз), ресурс убывает.
- **Гружён** → идёт к стройплощадке/складу, сдаёт груз.
Цикл повторяется, пока стройка не достроена / ресурс не кончился (тогда идёт к следующему источнику).

### 4.3 Привязь (leash)
Рабочая привязь шире боевой: **≈16м** (боевые отряды ≈7м). То есть рабочие курсируют ресурс↔стройка на бóльшую дистанцию, но не уходят далеко от башни. Добыча возможна только в **зоне сбора** вокруг башни (§2.1).

### 4.4 Прятки в башню
Команда «В башню»: рабочий забегает внутрь башни (невидим, неуязвим, снят с таргетов врага), пока не получит другую команду. Защита рабочих в осаде.

### 4.5 Ресурсы
Дерево / камень / железо (источники в комнатах) + золото (кувшины). Ресурсы копятся в башне-сторе / складе (расширяет кап). Дерево — основной строй-ресурс на текущем этапе.

## 5. Стройка — свободный силуэт (НЕ грид/сектора)

**Единая модель размещения** (важно для UE — это заменяет старую секторально-кольцевую стройку лагеря):
1. Игрок открывает меню «Стройка» (вкладка карточки отряда рабочих) → выбирает здание.
2. **Рука тащит призрак-силуэт** за курсором. ЛКМ-зажим фиксирует точку, **драг = поворот** (шаг 90°), отпускание ставит **стройплощадку**.
3. **Рабочие достраивают**: носят нужный ресурс на площадку, пока не наберётся норма.
4. Стройплощадка имеет HP — **скелеты могут сорвать стройку**, если ломают площадку.

**Каталог построек (текущий):**

| Здание | Что строится | Ресурс | Норма | HP площадки | Гейт |
|---|---|---|---|---|---|
| **Стена** | сегмент частокола (блокирует скелетов, разрушаем) | дерево | 4 | 35 | знание построек |
| **Сторожевая башня** | пост-лучник (конус обзора, стреляет по врагам) | дерево | 6 | 50 | знание построек + куплен отряд лучников |
| **Мост** | настил через пропасть (отдельный 2-кликовый flow) | «брёвна» ∝ длине | — | — | — (не гейтится) |

**Стены магнитятся** к соседним стенам (центр/края) — удобно строить лабиринты/линии. Башня-силуэт не магнитится.

**Гейт построек:** стена/башня заблокированы, пока игрок не получил **знание построек** — оно открывается запуском станка в Room11 (§8). До этого пункты серые с подсказкой.

## 6. Скелеты — поведение врагов

Все враги наследуют общую FSM: **APPROACH → WINDUP → STRIKE → COOLDOWN → APPROACH**. Базовые детали (vision, per-spawn variance, aggro-on-hit, LOD для производительности) — старый хэндофф §7. Ниже — типы и их отличия.

### 6.1 Базовый melee-скелет
- hp ≈30 (гибнет от одного парир-импульса / супер-дэша).
- Идёт к цели, у точки делает **AoE-выпад** (windup-телеграф → удар по площади).
- **Boids-избегание** соседей (не слипаются в точку), мягкий кап «не больше N на цель».
- Цели: палатки/гномы/башня/постройки (общая группа целей). Стены и палисад — «melee-only» цели (ranged-враги их игнорируют, melee ломают, чтобы пройти).
- В комнатном режиме комнаты заполнены бродячими скелетами (≈20 на комнату), агрятся на башню вблизи.

### 6.2 Скелет-лучник
- Дальний бой, **kite**: держит дистанцию [min..max], подходит/отступает, в зоне — стреляет.
- Снаряд — **баллистическая стрела с AoE-импактом** (летит по дуге в точку на земле, на приземлении бьёт по радиусу). Прицел «по площади» с разбросом — игрок/движущиеся гномы могут уклониться.
- Долгий windup = телеграф. Группа лучников (ArcherGroup) даёт синхронный залп в общую зону.

### 6.3 Скелет-гигант (melee-танк)
- hp ≈240, медленный, крупный, knockback-резист.
- **Фокус на башню** (приоритетная цель; room-гейт — агрится когда башня в его комнате).
- **Атака — супер-рывок (charge)** на башню вместо обычного удара: разгоняется и бьёт на контакте.
- **Уворот** от снарядов игрока (короткий рывок вбок).
- **«Тяжёлый»**: обычный таран башни его НЕ берёт — только супер-дэш + AoE/магия (см. §6.6).

### 6.4 Гигант-каменщик (ranged-танк, одиночный камень)
- hp ≈220, стоит на 25–35м, кидает **один тяжёлый AoE-камень** с телеграфом (кольцо зоны падения). «Тяжёлый» (super_dash_only).

### 6.5 **НОВЫЙ враг — Скелет-камнеметатель (Stone Thrower)**

Босс-«бомбардир». Третий вариант ranged-угрозы. **Это центральный новый враг периода — детально в §7.**

### 6.6 Урон башни по врагам (важное изменение)
- **Дэш-таран теперь бьёт ЛЮБОГО врага** (раньше только melee-скелетов). Контакт во время рывка → урон (обычный `≈100`, супер `≈250`), каждого враг раз за рывок.
- **«Тяжёлые» враги** (оба гиганта + оба каменщика) — в семантической группе «только-супер-дэш»: обычный таран и зона щита их **пропускают**, берут только **супер-дэш + AoE/магия**. (Не хардкод по типу — группа-маркер.)
- **Парирование/щит**: отражает снаряды, импульс-урон лёгким скелетам в куполе; тяжёлых не берёт.

## 7. Камнеметатель (Stone Thrower) — полная спека

**Концепция:** ranged-«бомбардир» размером с гиганта. В отличие от каменщика (1 валун) сыплет **россыпь ~30 мелких камней по дуге** в зону башни, **динамично перемещаясь** вокруг цели. Уворачивается от магии (хуже гиганта). При смерти — **мощный взрыв**. Выгодно убивать издалека.

### 7.1 Базовое
- **HP ≈240** (как melee-гигант). Размер капсулы как у гиганта (~0.5×4.4).
- knockback-резист 0.2 (танк, стрелы/слэм не сбивают).
- **«Тяжёлый»** (super_dash_only, §6.6): обычный таран не берёт, супер-дэш/AoE/магия — да.
- Приоритетная цель — **башня** (как оба гиганта), пока жива.
- Виден в тумане издалека (рассеивает дымку вокруг себя).

### 7.2 Залп (россыпь камней)
- **30 камней** высыпаются «росчерком» за ≈0.7с (стаггер по времени — эффект проходящего залпа, не мгновенной кучи).
- Точки падения — **равномерно в диске радиусом ≈7м** вокруг цели.
- **Каждая точка заранее отмечена на земле** кольцом-маркером (живёт примерно время полёта камня) — игрок видит, куда сядет каждый, и может увести юнитов.
- На WINDUP станок показывает **широкое кольцо всей опасной зоны** (телеграф).
- Один камень: AoE-радиус ≈1.6м, урон 5–9, гравитация выше обычной (крутая дуга).
- **Камни бьют ВСЁ в радиусе падения**: башню, гномов, палисад **и обычных скелетов** (френдли-фаер по своим).
- Темп: windup ≈0.7с, кулдаун ≈1.4с — **бьёт часто**.

### 7.3 Динамичное перемещение («танец»)
Не встаёт в фиксированное кольцо. Каждый цикл атаки:
- Выбирает **новую дистанцию выстрела** в диапазоне [16..34м] (не залипает на одной).
- Выбирает **сторону обхода** (по/против часовой) + случайный сдвиг угла.
- Подходит к выбранной дистанции **дугой** (радиальное движение к нужному радиусу + касательный обход), а не по прямой.
- Скорость ≈4 (быстрый для своего размера). Стреляет, дойдя до выбранного радиуса.

### 7.4 Уворот от магии (хуже гиганта)
Короткий рывок вбок от ближайшего снаряда игрока, НО «хуже»:
- Замечает снаряд позже (радиус ≈6 против гигантских 8).
- Реже готов (кулдаун ≈1.6с против 0.7).
- Прыгает недалеко (≈1.4м против ≈2.9). → single-target Искра часто всё равно попадает; AoE накрывает гарантированно.

### 7.5 Взрыв на смерти
- AoE-радиус ≈7м, урон ≈130 + нокбэк + тряска экрана.
- Бьёт башню, гномов, палисад **и скелетов**.
- Отсюда правило «убивай издалека» — вблизи взрыв накроет твою башню/отряд.

### 7.6 Таблица параметров (тюнинг)

| Параметр | Значение | Смысл |
|---|---|---|
| HP | 240 | как melee-гигант |
| move_speed | 4.0 | мобильный |
| attack_cooldown / windup | 1.4 / 0.7с | частый темп |
| дистанция выстрела | 16–34м (новая каждый цикл) | анти-залипание |
| volley_count | 30 | камней в залпе |
| volley_scatter_radius | 7м | разброс точек падения |
| volley_sweep_time | 0.7с | длительность росчерка |
| rock_aoe / rock_damage | 1.6м / 5–9 | один камень |
| dodge (radius/cd/dash) | 6 / 1.6с / 9×0.16с | уворот хуже гиганта |
| death_explosion (r/dmg/kb) | 7м / 130 / 8 | взрыв на смерти |
| strafe_factor / range_tol | 0.8 / 2.5 | обход / допуск дистанции |

## 8. Квест гномов-строителей

### 8.1 Гномы-строители (Room6) — квестгиверы
Поселение «Гильдия Камня». Диалог гейтится по прогрессу игрока (флаги):
- **Найм лучников** — доступен **после убийства камнеметателя** (§7). До этого объясняют: арсенал простреливает камнемёт. (Найм лучников перенесён сюда — раньше был у торговца в Room4.)
- **Знание построек** — дают, когда игрок **запустит станок-чертёжник** в Room11 (учёба «докажи, что умеешь с механизмами»).

### 8.2 Станок-чертёжник (Room11) — пазл «запусти механизм»
FSM: **IDLE → DEMO → INPUT → DONE**.
- Башня подошла → станок **показывает последовательность** (диоды вспыхивают по порядку).
- Игрок **повторяет искрой** по диодам в показанном порядке. Верно → диод фиксируется. Ошибка → сброс + повторный показ.
- Вся последовательность собрана → загорается **рычаг-пускач** → игрок дёргает → станок оживает (FX) → **знание построек открыто**.

Это **первое звено** большой цепочки восстановления нефтекачалки (§9). Пазл переиспользует язык «искра→диод→рычаг» (§3.3).

## 9. Нефтекачалка (Room8) — хребет геймлупа

> **Статус:** объект-якорь и стадия 1 реализованы; стадии 2–4 — в дизайне (описаны для имплементации).

### 9.1 Идея
Большой **мегабур** у поселения (Room8), который добывает «гномью нефть». **Сломан.** Игрок восстанавливает его **цепочкой квестов**, затем **защищает** во время добычи. **Накопил N нефти → победа матча.**

### 9.2 Цепочка стадий
1. **Лицензия строителя** — запусти станок Room11 (§8). *Реализовано.*
2. **Отстрой узлы / модули** — достраиваешь части качалки стройкой + чинишь механизмы. *Дизайн.*
3. **Запусти насос** — собрал → добыча пошла, нефть капает. *Дизайн.*
4. **Защити** — добыча стягивает орду, штурм на Room8; держишь оборону, копишь нефть. *Дизайн.*

### 9.3 Мегабур — объект
- Компактный (~2.6м): корпус + **раскалённое ядро-кольцо** + **бур-бит**, вгрызающийся в нефтяную жилу. В работе бит **крутится**, бьёт гейзер, трясёт экран.
- Состояния: **BROKEN → RESTORING → PUMPING**. В PUMPING добыча `oil_per_sec` копится к `oil_goal` (цель = победа).
- Гейт ремонта = «знание построек» (стадия 1).

### 9.4 Модульное кольцо (ключевая механика, **путь A**)
Модули — это **обычные постройки** (ставятся как стены: рука-силуэт → рабочие достраивают), которые **регистрируются на бур** в его зоне. НЕ слоты/сектора.
**Каждый модуль работает на ДВА фронта (добыча И оборона)** — раскладка = тактический баланс:

| Модуль | Добыча | Оборона |
|---|---|---|
| **Насос** | +нефть/с | шумит → притягивает больше врагов |
| **Турель** (реюз пост-лучника) | — | бьёт орду |
| **Охладитель** | держит бур в работе (без него перегрев → стоп) | — |
| **Броня** | замедляет добычу | +HP сектора качалки |

Первый насос = **момент запуска бура**. В осаде враги бьют модули/бур → восстанавливаешь под огнём.

# Часть B — Базовые системы (переносятся из прошлой итерации)

> Созданы в караван/лагерной итерации, но **остаются в текущем комнатном геймплее**: магия, ИИ скелетов, лучники, XP-прокачка, копейщики, супер. Здесь — сжато, но достаточно для имплементации. Полные тексты с UE-маппингами (GAS, Niagara, Behavior Tree и т.п.) — в `handoff_to_ue.md` §4–§7, §11–§12.

## B1. Магия — рука кастует из башни

Категории ввода руки: **1/2** физика, **3** Fireball, **4** Firestorm, **5** Mine Scatter, **Space** супер. ПКМ = каст активной категории (если рука пуста, курсор не над UI, заклинание не на КД, хватает маны). Снаряды вылетают **из вершины башни** (мана живёт в башне). Каждый каст рисует **наземное кольцо-телеграф** в точке удара ещё до полёта снаряда (для игрока, не для врагов).

**Искра (Spark)** — описана в §3.3. Зигзаг-снаряд, single-target урон, активирует диоды-пазлы. Это «рабочая» магия для механизмов.

**Fireball** (3→ПКМ): 2-фазная траектория — boost (0.18с, вверх+вперёд, своя гравитация 14, случайный sway) → homing (разгон до cap 55 м/с, на входе отклонение ±45° затем докрутка = «крюк»). Взрыв при <0.6м до цели. AOE с linear-falloff, **friendly fire включён** (бьёт и гномов/башню). Knockback. В эпицентре — BurnPatch.

| Ур | Dmg | Radius | CD | Mana | Burn/tick | Burn R |
|--:|--:|--:|--:|--:|--:|--:|
| 1 | 47 | 3.5м | 0.40с | 12 | 18 | 2.8м |
| 2 | 58 | 3.8м | 0.36с | 11 | 24 | 3.0м |
| 3 | 77 | 4.2м | 0.32с | 10 | 29 | 3.3м |
| 4 | 101| 4.5м | 0.28с | 9 | 37 | 3.5м |

**Firestorm** (4→ПКМ): серия 4–6 малых фаерболов (тот же Fireball, урезанный), интервал 0.15→0.11с, ложатся «ковром» в scatter-радиусе 2.8–3.2м; цель фиксируется в момент нажатия. Per-shot: ур1 40dmg/3.0м → ур3 64/3.5м. Mana 50–60 (атомарно — не хватило, серия не стартует), CD 1.6–2.0с.

**BurnPatch** (остаточный огонь обоих заклинаний): статичная зона, равномерный tick без falloff каждые 0.4–0.5с, длительность 2.5–3.5с, первый тик через интервал (не мгновенно), без knockback.

**Mine Scatter** (5→ПКМ): залп N мини-снарядов (Fireball-баллистика, dmg=0 на impact) → ставят мины в круге scatter. Мина FALLING→ARMING(0.5с)→ARMED (пульсирует), детонирует на любого Damageable рядом. **Friendly fire ON.**

| Ур | Mines | Scatter | Dmg | AOE | CD | Mana |
|--:|--:|--:|--:|--:|--:|--:|
| 1 | 11 | 5.0м | 30 | 1.8м | 4.0с | 40 |
| 2 | 12 | 5.5м | 36 | 2.0м | 3.6с | 42 |
| 3 | 13 | 6.0м | 44 | 2.2м | 3.2с | 44 |

**Мана:** в башне, max 100, реген 10/с (постоянный, не паузится боем). Атомарное списание.
**SpellSystem:** синглтон-источник истины (что unlocked + уровень). Каждый каст читает параметры через `get_current_level_data(id)`. Прокачка через ресурс **PAGE** в журнале. Fireball/Firestorm/Mine — с начала; meteor — заглушка-задел.

## B2. Базовый скелет (melee) — ИИ

FSM `APPROACH → WINDUP → STRIKE → COOLDOWN`:
- **APPROACH:** идёт к точке на approach-кольце (`target.pos + (cos,sin)×attack_range×0.85`, угол random per-цель → окружают кольцом, не линией). Дист ≤ `attack_range`(1.5м) → WINDUP.
- **WINDUP:** стоит 0.4с (0.1с если point-blank <1.05м), squash-поза-телеграф, медленно ползёт вперёд (1.5 м/с). Защёлкивает `_windup_target`. **Knockback в WINDUP → сброс в APPROACH** (игрок прерывает замах).
- **STRIKE** (транзитное): **AoE-урон в радиусе 1.95м** всем Damageable + self-lunge (физически летит на цель `dir×8 м/с×0.2с`).
- **COOLDOWN:** 1.0с ±15% → APPROACH.
- Без цели — **wander** (random точки 5–15м, скорость 1.2, отдых 1–3с; таймеры рандомизированы — стая «дышит»).

**Vision/таргетинг:** скан группы целей в `vision_radius`=12м каждые 0.4с. Приоритет: свободный гном (`target_load<2`) > здание > capped-гном. Forced-target (от волн) — только если vision пуст. **Persist-toward-last-known** (цель пропала, но жива → идёт к последней точке). **Aggro-on-hit:** на урон — немедленный rescan.

**Per-spawn variance** (±15–20%, один раз на спавне): hp 24–36 (база ≈30), dmg 6.4–9.6 (база 8), windup 0.32–0.48, move 3.4–4.6, cooldown 0.85–1.15. Цель — волна не автопилотится по запомненному ритму.

**Перф:** skel-skel физика выключена, вместо неё boids-avoidance (радиус 1.5м, сила 0.5×move). LOD по дистанции до камеры (NEAR/MID/FAR — даунскейл тика; для UE не критично, если перф тянет всех в полной детализации).

## B3. Скелет-лучник (базовый, не гигант)

Kite: `attack_radius` 8–14м (ближе — отступает 0.8×move, дальше — идёт 2.4 м/с), в зоне — WINDUP 0.7с → стрела → COOLDOWN 1.8с. Снаряд: баллистика, dmg 12–18, разброс прицела 1.5м, маска hostile (палисад **физически блокирует** низкие стрелы). HP 20. Не в группе melee-скелетов (нет boids/soft-cap — концентрированный огонь норма). Группа из 4 (ArcherGroup) даёт синхронный залп в общую зону.

## B4. Пост-лучник = «Сторожевая башня» (DefenderGnome)

Сторожевая башня (§5) строится как **пост-лучник** — та же боевая логика, что у стационарного защитника:
- **Конус зрения:** радиус 35м, полу-угол 45° (90° FOV); тело разворачивается на цель.
- **Alarm-канал:** на сигнал «атакован лагерь/база» разворачивается на атакующего даже вне конуса (persist 5с).
- **Бой:** в `attack_radius` (база 22.5м, +5 с апгрейдом) стоит/пятится (kite-апгрейд), стреляет каждые 1–2с. Сепарация от соседей (1.5м) + распределение огня (штраф за цель, по которой уже целятся — не бьют все в одного).
- **Стрелы:** баллистика speed 22, gravity 6, dmg 25–40.
- **Точность растёт с опытом:** `inaccuracy = 1.5 / (1 + shots/100)`. Per-инстанс, сброс на смерть → выгодно держать ветеранов живыми.

## B5. Squad XP и апгрейды

Отряд копит общий XP с убийств. **Орб (XpOrb)** на смерти врага: лежит (lifetime 60с) → касание союзника (башня/гном) → летит в базу (12 м/с) → `+XP`. Кривая `[50,120,250,500,1000]`. На уровень — выбор апгрейда (через журнал, level-gated). Примеры (лучники): `KITING` (пятится при близком враге), `LONG_DRAW` (+5м дальность). Гномы-собиратели активно подбирают орбы (приоритет над ресурсами — орбы исчезают).

## B6. Копейщики (pikeman) — мобильный отряд

`Squad` (логика: state + members[]). States: **HOLDING_POSITION** (кольцо вокруг точки), **ESCORTING_TOWER** (за башней с offset), **DEFENDING_CAMP** (патруль). Призыв: 5 юнитов за ресурсы. Stats: hp 30, detect 18м, attack_range 2.2м, dmg 22–32, cd 0.6–1.0с, move 2.2.
**Charge-cycle:** `READY → APPROACH` (разгон 0→3.5 м/с) `→ WINDUP → LUNGE` (рывок **22 м/с**, направление фиксируется → можно увернуться; на контакте dmg+knockback) `→ DRIFT` (0.25с) `→ RECOVERY` (0.7с уязвим). Окно recovery = расплата (скелет в point-blank бьёт быстрым windup'ом).
**Recall-wave (Q):** кольцо 20м от башни → юниты в зоне переключают escort/hold (toggle), вне зоны — «не услышал». **«Идти сюда»** (клик в UI → aim → ПКМ): strict-march, игнор боя до прибытия.

## B7. Супер-удар (ultimate башни)

Шкала копится с урона по врагам (1:1; в комнатной итерации порог большой — событие, не спам). Каст: **Space** → слоумо ×0.15 + **QTE-паттерн** → успех → прицел на земле → ПКМ → carrier-фаербол из башни в `точку+12м` → **burst** → **12 payload-фаерболов ковром** (круг 7м, каждый dmg ~47 / AOE 4м, random delay+jitter, нахлёст в центре). Провал QTE = −50% шкалы (не полный сброс). Отдельная категория ввода (не equip-slot).

## B8. Базовое поведение гнома-собирателя

Рабочие (§4) — специализация. Базовый собиратель: FSM `SEARCHING → COMMUTING_TO_PILE → COMMUTING_TO_BASE`; видит кучи ресурсов в 10м, берёт ближайшую не-claimed (claim не даёт всем бежать к одной), несёт в базу. Активно подбирает XP-орбы.

# Часть C — Легаси (караван/лагерь), заменено комнатным режимом

Системы прошлой итерации, **не являющиеся ядром текущего геймплея** (заменены анфиладой комнат + мобильной башней + нефтекачалкой), но остаются в коде:

- **Караван + лагерь (Camp/Tent/POI):** башня вела колонну палаток, разворачивалась в POI, гномы жили в палатках. → Теперь: гномы живут в домах комнат, башня мобильна без палаток-обоза.
- **Волны вокруг лагеря (WaveDirector + CombatGroup):** спавн волн в POI. → В комнатах: комнаты заранее заполнены бродячими скелетами + день/ночь-цикл WaveDirector (**ночь = штурм**, день = иди по квестам).
- **Build-zone постройки (колокол, brush-палисад, секторальный грид `build_grid`):** → Заменено **свободным силуэтом** (§5). Старый грид «умирает».
- **Ресурсы лагеря (единый инвентарь):** → ресурсы копятся в башне-сторе/складе, добыча рабочими в зоне башни.

Детали легаси — `handoff_to_ue.md` §2, §8, §9, §10.
